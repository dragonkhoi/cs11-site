Plug and Play VR SDKs

There are a few out-of-the-box SDKs that have a lot of easy pre-built solutions that can save you a lot of time. The one that I've worked with is called VRTK, which you can download through the Asset Store.

  • VRTK. VRTK has a ton of pre-built interactions like UI, teleporting, grabbing, and more. It's a little weird to get it set up, but afterwards, it's pretty intuitive. I would download it from the asset store and then play around with the sample scenes.

  • Oculus Developer Packages. Lots of good stuff from Oculus like avatars with hands and other social platform stuff, ambisonic/3D sound, and various demos.
  • NewtonVR. Haven't used this, but it has a lot of good physics interactions.
  • Photon Multiplayer. Multiplayer is a nightmare, but Photon makes it a little more manageable. You can have up to 20 concurrent users for free.

Particles and Shaders

Particles can make an experience look really good. They add ambience, delight, and feedback to any experience. Note: don't overuse particles on the Oculus Go, or it'll slow down.

Collaboration

To collaborate on your Unity project, you'll want to use version control.
Here are some resources you can use:

  • Git! Git works with Unity files. For those unfamiliar with Git, you can learn here or come to my office hours:
    • If you are new to Git, I recommend using the desktop app instead of the command line.
    • Use the Unity .gitignore to avoid pushing unnecessary files.
    • Have only one person work on a scene at a time (other people can be making prefabs, writing code, etc, in other scenes). Unity scene merge conflicts are impossible to resolve.
    • Large files will have to be pushed through Git LFS (Large File System).
  • Github for Unity. I've personally never used this, but it looks super useful! You can download it through the asset store, and it will seamlessly integrate with your project.
  • Unity Collab. Probably don't use this unless I can get some pro licenses for our class.
