Assignment 3: Destroying Moving Blocks

In this assignment, to be completed in class, you will make the cubes move and explode.

Lightsaber to Hand

In your project, go into Oculus --> VR --> Prefabs and drag the TrackedRemote prefab onto LeftHandAnchor and RightHandAnchor to child it.

Click the TrackedRemote under the LeftHandAnchor, and set it to L Tracked Remote in the inspector.

Move your lightsaber (the cylinder we made last week ) under the OculusGoControllerModel.

Make sure your lightsaber has a Capsule Collider with Is Trigger checked, a Rigidbody with Use Gravity UNchecked and Is Kinematic checked, and a SaberControl script.

Adjust your lightsaber's transform so it looks aligned with the controller. My values are:
Position: X: 0 - Y: 0.68 - Z: 1.88
Rotation: X: 70 - Y: 0 - Z: 0
Scale: X: 0.1 - Y: 2 - Z: 0.1

In your project, create a new folder called xPrefabs and drag the lightsaber from the scene into the project view.

Prefabs are GameObjects that have basically been saved into your project with all of their settings. You can easily reuse them this way, or create new GameObjects using these as templates!

Grab the prefab you just created from the project view and drag it under the RightHandAnchor's OculusGoControllerModel. Notice how it has all the same settings as the other one!

Bonus : If you adjust one Lightsaber's Scale and click APPLY (not shown in the above photo, but under the object name where it says Prefab, all the way on the right), the other one will update too!

Deactivate the lightsaber on the hand that you're not going to use. My Oculus Go has right handedness on, so I'll turn off the left hand lightsaber. Deactivating an object means it will not be visible, its colliders don't affect other ones, and its scripts don't run.

Usually, Oculus will handle handedness disabling for us, but our SaberControl code will make the two lightsabers destroy each other. We can fix that later using a Tag check.

Move the Cubes

Create a new script called MoveTransform and put it in your "xScripts" folder.

Create two variables: Vector3 moveDir and float speed and make them private and use [SerializeField] for both variables.

Private variables are only accessible within the class, which keeps it safe from hard-to-track external modifications.
It is a standard coding convention to keep variables private unless it is necessary for public access.
[SerializeField] exposes the variable value in the inspector.

In the Update loop type translate.Translate(moveDir * speed);.

HW QUESTION: What do you think Translate does? What are moveDir and speed?

Select the CubeCircle object. Attach the MoveTransform script to the CubeCircle object by dragging it onto the inspector.

On the MoveTransform script, set the Move Dir Z to -1. Set the Speed to 0.05. Hit Play.

Move the CubeCircle object back on the z-axis so it is far away from the camera.

Hit play.

Go into the CubeCircle object and make more cubes that are spread out, but make sure they are narrow. Adjust the speed to your liking, 0.05 is normal, but 0.1 is harder.
